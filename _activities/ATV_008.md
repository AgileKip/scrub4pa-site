  - Especificar os **requisitos funcionais** e **requisitos não funcionais** do software. Se um modelo de processo foi especificado anteriormente, este deveria ser usado para ajudar na identificação das funcionalidades do software.

  - Elaborar o **modelo de domínio** do software (caso os requisitos representem uma evolução de alguma ferramenta desenvolvida em outro projeto, deve-se atualizar o modelo de classes da mesma).

  - *Se pertinente ao projeto*, elaborar **diagramas complementares**, como de atividades, sequência, estados, de colaboração ou outros.
